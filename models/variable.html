class metacatalog.models.variable.Variable(**kwargs)¶

Model to represent variables. The variable is any kind of oberservation, that can be represented by one data type. metacatalog does not take the definition of variables too strict. It is however common to keep variables as atomic as possbile.

However, technically, you can also create a new variable that describes a combined data type and reference a newly created table via DataSource <metacatalog.models.DataSource>. This can make sense if in the scope and context of the metacatalog installation a sensor like a Decagon 5TE always records three parameters at a time like Temperature, Moisture and Conductance. That can be implemented as a new ‘5TE’ variable and the datasource would point to a table containing all three measurements. Note that this should not be common practice and will make your metadata unusable in other contexts.
